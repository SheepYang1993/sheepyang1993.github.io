使用 Hexo + Github Pages 搭建个人博客

发表于 2016-08-31   |     |   阅读次数

前言

一直想要搭建个人博客，网上搜了下资料，目前比较流行的是使用Hexo + GithubPage搭建静态网站。花费了一天时间把博客搭好了，现在记录整理下搭建过程以及所遇到的坑。想要拥有个人博客的朋友可以参照下，花费半小时搭建属于自己的博客。

一、 Git工具、Node.js安装

首先是搭建Hexo，需要安装 Git工具（Git安装教程） 以及 Node.js（Node.js安装教程）。安装完成后，在桌面空白处右键打开 git bash 命令行工具。

二、 Hexo的安装及使用

打开 Git Bash，执行以下命令开始安装Hexo：npm install -g hexo

安装成功后，新建一个文件夹用来存放你的博客以及相关配置。在 git bash 中输入 cd path 进入到新建的文件夹， path 是新建文件夹的路径。

执行： hexo init 进行初始化

接着执行以下命令生成静态页面至public目录： hexo g

完成以上步骤后，Hexo就算是搭建好了。

接下来我们就可以在本地预览我们的博客了！
执行 hexo s 启动服务

打开 https://localhost:4000 就可以看到效果了。如果打不开，可能是4000端口被占用了，那我们可以换个端口号启动服务，执行 hexo s -p 5000 打开 https://localhost:5000 就可以看到效果了。按 Ctrl + C 可以停止服务。

到这步为止，我们的博客就可以使用啦！那怎么发布博文呢？
首先执行： hexo new “my new post” ，其中 my new post 为文章标题。在 /source/_posts/ 目录下就可以看到新创建的md文件了。文章的编写需要使用 markdown 语法编写文章， markdown使用方法、markdown编辑工具

每次编辑、修改文章后执行: hexo clean ， hexo g 清除缓存重新编译，启动服务后可在本地查看博客。

三、 修改Hexo主题、配置

Hexo静态博客想要安装主题也容易，在Github上找到你喜欢的主题，然后执行类似命令： git clone https://github.com/heroicyang/hexo-theme-modernist.git themes/modernist

这边有一些常用的主题大家可以拿去用 Hexo主题

下面以本人博客主题Yilia为例,首先找到Yilia Github地址,点击右上角 Clone or download 复制地址到粘贴板

在 git bash 中执行： git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

执行后yilia主题就下载在themes目录了

接着配置Yilia主题，打开 _config.yml 文件（根目录下的 _config.yml ，不是themes文件下的 _config.yml ）将themes修改为yilia，与themes文件夹下yilia目录同名。

之后依次执行 hexo clean ， hexo g ， hexo s 重新编译生成静态页面，启动本地服务。打开 http://localhost:4000 就可以预览效果啦！

接下来就是设置各种相关配置了，比如 姓名/头像/友情链接 等等。
先打开根目录下的 _config.yml 进行修改

再打开themes/yilia主题下的_config.yml进行修改

在配置完后，在这边也遇到了个坑，就是个人头像下面的小房子点击无反应，还有github、weibo的外链找不到图片资源，外加没有动画效果显示。最后在Yilia主题的issues下找到了解决方法，分别是issues277和issues288。如果不想看具体怎么解决的话，下载Yilia主题修复包，解压后覆盖替换掉 themes/yilia 目录下的 /source 、 /layout 、 /languages 三个文件夹。

四、 Hexo 部署至 Github Pages

终于有了自己的博客啦~ 不过现在你只能在本地浏览自己的博客，通过外网是不能直接访问到你的博客的。要在外网访问，我们必须拥有两个东西： 域名 和 网络空间 。域名大家可以自行到万网上去申请，最便宜的域名才2块钱一年。网络空间我们使用Github Pages，它为我们提供了300M的免费空间。

接下来讲讲怎么把Hexo部署到Github Pages上，首先你要注册一个Github帐号（如何注册Github帐号）。注册过程中需要谷歌验证，如果你显示不了谷歌验证，你懂的那就是被“墙”了，使用vpn翻墙解决吧。

登陆Github，创建一个新的仓库

仓库的名字是 xxx.github.io ， xxx 是你的账户名，比如我这边账户名是 SheepYang1993 ，那么仓库名就是 sheepyang1993.github.io

完成该步骤你就可以访问 Github Pages 了，下面我们要把本地的Hexo博客部署到 Github Pages 上。

首先我们打开 cmd 命令行工具，输入 ping xxx.github.io 命令， xxx 是你Github的账户名，获得 xxx.github.io 的ip地址

打开根目录下的 _config.yml ，设置你要部署的Github地址

打开根目录下的/sourcewen文件夹，新建一个文件名为“CNAME”的文件，注意没有文件后缀名！里面写入你申请的域名。

接着来到域名的后台管理，我这边是通过万网注册的域名，直接登陆后台。点击 “解析”

输入我们之前ping到的ip地址

完成以上步骤后，和之前本地发布文章一样，创建修改文章后，依次执行 hexo clean 、 hexo g 、 hexo d，执行 hexo d 命令的时候会要求你输入Github帐号密码，输入后等待片刻，网站就发布成功啦~
